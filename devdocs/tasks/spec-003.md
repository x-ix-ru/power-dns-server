# Спецификация задачи 003: Проверить и усилить init.sh

## Описание

Проверить скрипт инициализации `init.sh` и убедиться в корректности и надёжности.

## Полный контекст

Текущий `init.sh`:
1. Проверяет наличие БД (`/var/lib/powerdns/pdns.db`)
2. Если нет — создаёт таблицы SQLite (domains, records, supermasters, comments, domainmetadata, cryptokeys, tsigkeys)
3. Ставит права `pdns:pdns` на БД
4. `exec pdns_server --daemon=no --write-pid=no`

Скрипт уже работает корректно. Нужно проверить edge cases:
- Запуск при уже существующей БД (idempotent) ✅ — уже реализовано
- Права на файл БД ✅ — `chown pdns:pdns` уже есть
- Передача сигналов через `exec` ✅ — уже использует `exec`
- Конфликт `--daemon=no` — pdns.conf уже содержит `daemon=no`, но явная опция CLI не вредит

## Технические детали

### Что проверить

1. **Idempotent init** — `if [ ! -f "$DB" ]` блок корректен. При повторном запуске с volume: контейнер видит существующую БД, пропускает создание, идёт к `exec pdns_server`. ✅

2. **Права доступа** — `chown pdns:pdns "$DB"` ставится после создания. Каталог `/var/lib/powerdns` уже имеет `chown -R pdns:pdns` в Dockerfile. ✅

3. **exec pdns_server** — корректно заменяет процесс shell на pdns_server. Сигналы (SIGTERM от docker stop) будут переданы напрямую. ✅

4. **Дубликаты опций** — `exec pdns_server --daemon=no --write-pid=no` дублирует pdns.conf. Это не ошибка, но можно убрать опции из exec, оставив только в конфиге. Или оставить как есть для надёжности.

### Что может быть улучшено

Минимальные правки:
- Добавить `#!/bin/sh` проверку (уже есть)
- Можно убрать `--daemon=no --write-pid=no` из exec, так как они уже в pdns.conf
- Добавить `set -euo pipefail` для большей строгости (но `sh` в Alpine это `ash`, поддерживает только `set -eu`)

### Вердикт

Скрипт практически корректен. Минимальные правки:
- Убрать дублирующие CLI опции из `exec` (уже в pdns.conf)
- Или оставить как есть (defensive coding)

**Рекомендация:** оставить как есть. Скрипт корректен.
